# VuePress 文档格式规范

## 1. Frontmatter 规范

### 必填字段

```yaml
---
title: 文档标题              # 必填：文档标题
description: 文档描述        # 必填：1-2句话描述内容
date: YYYY-MM-DD           # 必填：创建或更新日期
tags: [标签1, 标签2]        # 必填：至少一个标签
---
```

### 可选字段

```yaml
---
categories: [分类]           # 可选：文档分类
order: 10                   # 可选：排序权重（数字越小越靠前）
author: 作者名              # 可选：作者信息
---
```

### 命名规范

- **title**：简明扼要，使用中文
- **description**：一句话概括，不超过 50 字
- **date**：格式为 YYYY-MM-DD
- **tags**：使用下划线或驼峰命名，如 `api_rest` 或 `apiRest`
- **categories**：主要分类，如 `API`、`教程`、`示例`

## 2. 页面结构规范

### 标准结构

```markdown
---
frontmatter
---

<h1 style="text-align: center;">标题</h1>
<el-divider border-style="dashed">
    <div> ~ 副标题 ~ </div>
</el-divider>

## 概述

## 核心功能

## 使用说明

## 示例

## 注意事项

## 总结

<el-divider border-style="dashed">
  <div> · End · </div>
</el-divider>
```

### 标题层级

- **H1**：文档标题，只出现一次
- **H2**：主要章节
- **H3**：子章节
- **H4**：详细说明

**注意**：不要跳级使用（如 H1 后直接用 H3）

## 3. VuePress 自定义容器规范

### 使用场景

| 容器类型 | 适用场景 | 示例 |
|---------|---------|------|
| `tip` | 有用的提示信息 | 技巧、优化建议 |
| `warning` | 需要注意的警告 | 注意事项、潜在风险 |
| `danger` | 危险操作或重要警告 | 数据删除、不可逆操作 |
| `info` | 补充说明信息 | 背景信息、补充说明 |
| `details` | 可折叠的详细信息 | 代码详情、配置选项 |

### 使用格式

```markdown
::: tip 提示
提示内容
:::

::: warning 警告
警告内容
:::

::: danger 危险
危险操作
:::

::: info 信息
信息内容
:::

::: details 标题（可选）
可折叠的内容
:::
```

### 最佳实践

- 简短提示使用 inline 方式：`::: tip 提示 :::`
- 长内容使用 block 方式
- 嵌套使用时保持清晰的缩进

## 4. 表格规范

### 基本格式

```markdown
| 表头 1 | 表头 2 | 表头 3 |
|--------|--------|--------|
| 内容 1 | 内容 2 | 内容 3 |
| 内容 4 | 内容 5 | 内容 6 |
```

### 对齐方式

```markdown
| 左对齐 | 居中 | 右对齐 |
|:-------|:----:|-------:|
| 内容   | 内容 | 内容   |
```

### 表格命名

每个表格应该有一个标题：

```markdown
**参数说明表**

| 参数名 | 类型 | 说明 |
|--------|------|------|
| param1 | string | 参数说明 |
```

### 表格内容规范

- 表头应该清晰描述列内容
- 内容应该简洁明了
- 使用 `N/A` 表示不适用
- 使用 `-` 表示无默认值
- 使用 `✓` 和 `✗` 表示是/否

## 5. 代码块规范

### 语言标识

```markdown
```cpp
// C++ 代码
```

```python
# Python 代码
```

```bash
# Shell 命令
```

```json
// JSON 数据
```

### 代码说明

每个代码块后应该有说明：

```markdown
```cpp
class MyClass {
public:
    void myFunction() {
        // 实现
    }
};
```

**代码说明**：这段代码展示了如何定义一个类...
```

### 代码高亮

使用高亮突出关键部分：

```markdown
```cpp
class MyClass {
public:
    void myFunction() {  // 高亮的方法
        return true;     // 高亮的返回语句
    }
};
```
```

## 6. 图片规范

### 图片格式

```markdown
![图片说明](./image.png)

*图 1.1：图片详细说明*
```

### 图片标识

使用 emoji 标识不同类型：

- 📊 图表
- 🎨 界面截图
- ⚙️ 配置界面
- 🖼️ 效果展示
- 📱 移动端界面
- 💻 桌面端界面

### 图片命名

- 使用小写字母和连字符：`feature-overview.png`
- 有意义的名称：`installation-screen.png`
- 避免特殊字符和空格

## 7. 列表规范

### 无序列表

```markdown
- 项目 1
- 项目 2
  - 子项目 2.1
  - 子项目 2.2
- 项目 3
```

### 有序列表

```markdown
1. 步骤 1
2. 步骤 2
   1. 子步骤 2.1
   2. 子步骤 2.2
3. 步骤 3
```

### 复选框列表

```markdown
- [x] 已完成项 1
- [x] 已完成项 2
- [ ] 待完成项 3
```

### 列表分组

使用标题分组：

```markdown
**功能列表**：
- ✅ 功能 1：功能说明
- ✅ 功能 2：功能说明
- ⏳ 功能 3：待开发功能

**注意事项**：
1. 注意点 1
2. 注意点 2
3. 注意点 3
```

## 8. 链接规范

### 内部链接

```markdown
[链接文本](./other-doc.md)
[链接文本](#section-id)
```

### 外部链接

```markdown
[官方文档](https://docs.example.com)
[GitHub 仓库](https://github.com/user/repo)
```

### 链接文本

- 使用有意义的链接文本
- 避免使用 "点击这里"
- 链接文本应该描述目标内容

**好的例子**：
- [API 参考文档](./api.md)
- [安装指南](./installation.md)

**不好的例子**：
- [点击这里查看](./api.md)
- [更多](./more.md)

## 9. 强调和标记规范

### 粗体

```markdown
**重要内容**
```

**使用场景**：
- 强调重要信息
- 突出关键词
- 标记警告

### 斜体

```markdown
*强调内容*
```

**使用场景**：
- 强调术语
- 引用说明
- 语气强调

### 删除线

```markdown
~~已废弃内容~~
```

**使用场景**：
- 标记已废弃的功能
- 表示已删除的内容
- 对比新旧内容

### 行内代码

```markdown
`code`
```

**使用场景**：
- 函数名：`myFunction()`
- 变量名：`myVariable`
- 文件名：`config.json`
- 命令：`npm install`

## 10. 引用规范

### 引用文本

```markdown
> 引用的文本内容
>
> 第二段引用内容
```

### 引用嵌套

```markdown
> 外层引用
>
> > 内层引用
```

### 引用场景

- 引用名言或格言
- 引用官方文档
- 引用其他文档内容

## 11. 分隔线规范

### 标准分隔线

```markdown
---
```

### Element Plus 分隔线

```markdown
<el-divider border-style="dashed">
    <div> ~ 分隔文字 ~ </div>
</el-divider>
```

**使用场景**：
- 页面标题下方
- 主要章节之间
- 文档结尾

### 样式选项

- `border-style="solid"`：实线
- `border-style="dashed"`：虚线（推荐）
- `border-style="dotted"`：点线

## 12. 格式一致性规范

### 标点符号

- 中文使用中文标点：，。；：？！
- 英文使用英文标点: ,.;:?!
- 代码和英文引号使用英文引号：" ' `
- 中文引号使用中文引号：" " ' '

### 空格规范

- 中文与英文之间加空格：`使用 Python 编程`
- 中文与数字之间加空格：`版本 1.0.0`
- 运算符两边加空格：`a + b = c`
- 冒号后加空格：`key: value`

### 换行规范

- 段落之间空一行
- 标题前后空一行
- 代码块前后空一行
- 表格前后空一行

## 13. 特殊场景规范

### API 文档

- 使用 `api-doc-template.md` 模板
- 包含接口信息、请求参数、响应数据
- 提供多语言示例
- 包含错误码说明

### 教程文档

- 使用 `tutorial-template.md` 模板
- 包含环境准备、快速开始、实战教程
- 提供详细的步骤说明
- 包含常见问题和解答

### 示例文档

- 代码示例应该完整可运行
- 提供输入和预期输出
- 添加详细的注释
- 说明代码的目的和原理
