# VuePress 文档格式化 Skill

## Skill 描述

这是一个专业的 VuePress 文档格式化工具，用于统一和美化 Markdown 文档格式，提升文档的可读性和专业性。

## 何时使用此 Skill

在以下情况下使用此技能：

- 用户需要统一 VuePress 项目的文档格式
- 用户需要美化已有的 Markdown 文档
- 用户需要为文档添加 frontmatter 元数据
- 用户需要优化文档结构和可读性
- 用户需要批量格式化多个文档
- 用户需要遵循特定的文档风格规范

## 核心功能

### 1. Frontmatter 标准化

为每个文档添加完整的 frontmatter：

```yaml
---
title: 文档标题
description: 简短描述文档内容（1-2句话）
date: YYYY-MM-DD
tags: [标签1, 标签2, 标签3]
categories: [分类]
order: 排序权重（可选）
---
```

### 2. 页面头部装饰

添加统一的页面头部样式：

```markdown
<h1 style="text-align: center;">文档标题</h1>
<el-divider border-style="dashed">
    <div> ~ 副标题或装饰性文字 ~ </div>
</el-divider>
```

### 3. 内容组织结构

使用清晰的章节划分：

- **概述/简介**：简要介绍文档内容
- **核心功能**：列出主要功能点
- **使用说明**：详细的使用步骤
- **示例代码**：实用的代码示例
- **注意事项**：重要的注意点
- **总结**：文档总结

### 4. VuePress 自定义容器

使用 VuePress 提供的自定义容器增强文档：

```markdown
::: tip 提示
有用的提示信息
:::

::: warning 警告
需要注意的警告信息
:::

::: danger 危险
危险操作或重要警告
:::

::: info 信息
补充说明信息
:::

::: details 详细信息
可以折叠的详细信息
:::
```

### 5. 表格美化

使用表格整理信息，并添加适当的表头：

```markdown
| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| param1 | string | 是 | - | 参数说明 |
| param2 | number | 否 | 0 | 参数说明 |
```

### 6. 代码块优化

为代码块添加语言标识和说明：

```markdown
```cpp
// C++ 代码示例
class MyClass {
public:
    void MyFunction() {
        // 实现
    }
};
```

**代码说明**：这段代码展示了...
```

### 7. 图片优化

为图片添加说明和适当的标识：

```markdown
![图片说明](./image.png)

*图 1.1：图片详细说明*
```

使用 emoji 标识不同类型的图片：
- 📊 图表
- 🎨 界面截图
- ⚙️ 配置界面
- 🖼️ 效果展示

### 8. 列表优化

使用清晰的列表结构：

```markdown
**功能列表**：
- ✅ 功能 1：功能说明
- ✅ 功能 2：功能说明
- ⏳ 功能 3：待开发功能

**注意事项**：
1. 注意点 1
2. 注意点 2
3. 注意点 3
```

### 9. 链接优化

使用有意义的链接文本：

```markdown
- [官方文档](https://docs.unrealengine.com)
- [相关教程](./相关文档.md)
- [代码示例](./examples/example.cpp)
```

### 10. 文档结构模板

标准的文档结构：

```markdown
---
title: 文档标题
description: 文档描述
date: 2024-03-12
tags: [标签1, 标签2]
categories: [分类]

<h1 style="text-align: center;">文档标题</h1>

## 使用说明

### 步骤 1

...

```

重要的注意事项

<el-divider border-style="dashed">

</el-divider>

## 工作流程

### 步骤 1：分析现有文档

读取现有文档，分析：
- 是否有 frontmatter
- 内容结构如何
- 使用了哪些格式化元素
- 需要改进的地方

### 步骤 2：应用标准格式

按照标准格式模板，对文档进行：
- 添加或更新 frontmatter
- 添加页面头部装饰
- 优化章节结构
- 使用 VuePress 自定义容器
- 美化表格和代码块
- 优化图片和列表

### 步骤 3：验证格式

检查文档是否符合：
- Markdown 语法正确
- VuePress 语法正确
- 格式一致性
- 可读性良好

### 步骤 4：保存文档

使用 `replace_in_file` 或 `write_to_file` 保存格式化后的文档

## 最佳实践

### 1. 保持一致性

- 所有文档使用相同的 frontmatter 结构
- 标题层级保持一致（H1 -> H2 -> H3）
- 使用统一的分隔线样式

### 2. 增强可读性

- 使用自定义容器突出重要信息
- 为代码块添加说明
- 为图片添加详细的图注
- 使用表格整理复杂信息

### 3. 优化导航

- 添加合理的目录结构
- 使用内部链接方便跳转
- 在文档末尾添加相关文档链接

### 4. 语义化标记

- 使用 `**粗体**` 强调重要内容
- 使用 `*斜体*` 表示强调
- 使用 `~~删除线~~` 表示已废弃内容

## 常见场景

### 场景 1：格式化单个文档

用户请求："帮我格式化这个文档"

**执行步骤**：
1. 读取文档内容
2. 分析现有结构
3. 应用标准格式
4. 保存文档

### 场景 2：批量格式化文档

用户请求："批量格式化文件夹目录下的所有文档"

**执行步骤**：
1. 列出目录下所有 .md 文件
2. 按优先级排序（大文件优先）
3. 逐个读取并格式化
4. 保存格式化后的文档

### 场景 3：创建新文档模板

用户请求："创建一个 API 文档模板"

**执行步骤**：
1. 使用 assets/templates/api-doc-template.md
2. 根据需要调整模板内容
3. 保存到目标位置

## 工具使用

### 推荐工具

- **list_files**：列出目录下的文档
- **search_file**：查找特定文件
- **read_file**：读取文档内容
- **replace_in_file**：修改文档（适用于小改动）
- **write_to_file**：写入新文档（适用于大改动）
- **read_lints**：检查语法错误

### 工具使用技巧

1. **批量处理**：使用 list_files 获取文件列表，然后批量处理
2. **增量修改**：对于小改动，使用 replace_in_file
3. **完全重写**：对于大改动，使用 write_to_file
4. **错误检查**：修改后使用 read_lints 检查错误

## 注意事项

### 1. 不要删除原有内容

- 保留原有的实质性内容
- 只调整格式，不改变原意
- 如需删除内容，先与用户确认

### 2. 保持代码示例准确性

- 不要修改代码示例的逻辑
- 只调整代码块的格式和注释
- 确保代码可以正常运行

### 3. 尊重用户的格式偏好

- 如果用户有特定的格式要求，优先遵守
- 可以建议改进，但不要强制修改
- 与用户确认后再进行重大格式变更

### 4. 处理特殊情况

- 某些文档可能需要特殊的格式（如技术规范、API 文档）
- 尊重这些特殊需求，灵活调整格式
- 如果不确定，询问用户的偏好
